\section{Evaluation}
\label{sec:evaluation}

The evaluation combines an address-level production evidence audit, controlled
implementation checks, and a public-testnet enforcement execution. The first
pins the active Gnosis committee and computes both its supported floor and the
evidence needed to change it. Controlled profiles exercise the certificate
branches and check the reference implementation. The Chiado pilot tests a
machine-readable evidence-to-penalty transition.

We ask what the production record certifies; how activation order and
acquisition mechanism change fixed-resistance costs; whether local hardening
misses coordinated covers; and whether a public testnet penalty can execute
without being transferred to production.

The controlled checks use deterministic Python instances; random lattice
checks use seed \(20260714\). All ten recorded timing runs are retained. The
scaling machine ran Python~3.11.1 on an Intel Core i7-13650HX with 16.85~GB
physical memory. The enforcement implementation uses Rolling Shutter~v1.4.4,
Solidity~0.8.28, Hardhat, Node.js~22, and Gnosis Chiado. Seven validator and
seven Keyper processes run on one host. The controlled environments check
implementation paths and complexity behavior; the production audit supplies
real committee identities but does not infer unobserved member resistance.

\subsubsection{Activation paths and mechanism scope}

The activation ladder fixes ten equal member weights, threshold \(t=0.5\),
five cheap members of resistance one, and five seed members of resistance ten.
Parameter \(r\) is the number of seed acquisitions required before the cheap
path can begin. Resistance-only threshold cover remains five, while exact
activation cover is
\[
   \operatorname{ACR}_{R,\tau^{(r)}}(\varnothing)
   =10r+(5-r)=5+9r.
\]
The mechanism stress test uses the same separation principle with \(k\) seed
members and two core members. Sequential acquisition must pay for the seeds;
an all-or-nothing package purchases the core together.

\subsubsection{Defensive allocation and interaction order}

The allocation instance has four certified minimal covers over six bottleneck
members and two high-weight decoys. Every minimal cover starts at cost six.
For integer budget \(H\), the bottleneck rule solves the fixed-family max--min
problem. Two local baselines allocate to the cheapest current resistance or
cycle by nonincreasing member weight.

At \(H=4\), both local rules spend outside at least one binding cover and
leave the certificate at six; the bottleneck allocation reaches eight. At
\(H=10\), both local rules return nine while the optimum returns 11. These are
certificate decisions, not loss estimates: the comparison is valid only
because the controlled ledger certifies the outer family.

Exact rational-arithmetic checks cover 18 pure \(k\)-way constructions. Every
nonempty proper M\"obius coefficient is zero, and only the full-order
coefficient is nonzero. Removing it underpredicts the full-defense value
by factors up to 100 in the tested family. In a separate seeded study of 100
monotone four-of-seven certificate functions, singleton greedy is often
competitive on small random instances, but the fixed decoy construction
reaches a gain ratio of approximately \(5.0\times10^{-5}\). The random
behavior therefore does not repair the worst-case impossibility result.

Equal means do not imply equal certificates: at zero exposure the values range
from 16 to 40 solely because the lower tails differ. Initial exposure reduces
every tested certificate because fewer remaining shares are needed. For the
heavy-tail profile at zero exposure, raising the threshold from three to six
shares changes the certificate from \(10\) to \(18\), \(29\), and \(45\).
These monotonicities are checked over all 48 grid rows.

The solver results track the \(2^n\) state count. Median time grows from
0.000683 seconds at \(n=8\) to 7.495 seconds at \(n=18\); the latter case has
262,144 states. The wide \(n=18\) timing variation is retained rather than
trimmed. This supports the exponential limitation, not large-committee exact
scalability. The uniform threshold-cover special case avoids enumeration by
sorting, while integer-scaled activation inputs may use the
pseudo-polynomial dynamic program.

The Chiado exporter requires seven BLS shares, seven native signatures, the
aggregate key, and four-share reconstruction. Its EIP-712 attestation executes
forfeiture; the certificate binds nine artifacts, and a live verifier checks
eleven receipts and state. This is an executed evidence-to-enforcement path,
not independent governance or native on-chain BLS verification.

Production and Chiado are not pooled: their members, environment, penalties,
and evidence procedures differ. Dune activity is calibration, not a worst-case
\(\overline V(x)\). Production remains on the threshold-cover branch of
Theorem~\ref{thm:information-boundary}; using AC requires member-specific,
mechanism-matched activation evidence.

\subsection{Limitations and Reproducibility}

The audit does not estimate bargaining or actual willingness to collude.
Approximate activation inputs remain unsafe near path boundaries, and general
exact search does not scale. Production evidence also lacks the positive
member floors and worst-case value cap needed for a no-profit claim.

The artifact includes the pinned set, seven-row ledger, member-gap CSV, audit
JSON, archival-RPC verifier, and Chiado certificate. Deterministic results run
offline; timing provenance and live-chain steps appear in
Appendices~\ref{app:extended-reproduction} and~\ref{app:chiado-live-verification}.
